Features

Everything you need to go from Word document to AI-ready test data.

📄

DOCX Parsing

State-machine parser extracts sections, test cases, prompts, expected behaviors, pass/fail criteria, and preconditions from structured Word documents.

📊

Dual Output

Generates both hierarchical .json and flat .jsonl (one test per line) for direct upload to Azure AI Foundry evaluation.

✅

45+ QA Checks

Embedded test suite validates structure, completeness, field integrity, AI Foundry compatibility, prompt quality, and cross-reference consistency.

🔍

Delta-Diff Validator

Compares source DOCX against JSON output to detect prompt drift, missing entries, extra entries, and keyword loss with similarity scoring.

🔄

Regression Detection

JSON-to-JSON comparator detects drift between conversion runs -- metadata, prompts, behaviors, and structural changes.

✍️

Dual Mode

Works as a CLI tool (python docx_to_json_tool.py convert) and as a Python library (from docx_to_json_tool import convert_docx).

Quick Start

Up and running in under a minute.

Terminal 
# Clone the repository
git clone https://github.com/arjunghosh/docx-to-json-test-case-converter.git
cd docx-to-json-test-case-converter

# Install dependencies
pip install python-docx pytest

# Convert your test cases
python docx_to_json_tool.py convert your_test_cases.docx

# Or use as a Python library
from docx_to_json_tool import convert_docx
result = convert_docx("your_test_cases.docx")
print(result["status"])      # "PASS" or "FAIL"
print(result["jsonl_path"])  # path to .jsonl output

Frequently Asked Questions

Common questions about the tool, format, and integration.

How do I convert a Word document to JSON for AI testing?
Install the dependency with pip install python-docx, then run python docx_to_json_tool.py convert your_test_cases.docx. The tool parses structured test cases and outputs validated JSON and JSONL files ready for Azure AI Foundry or any AI agent evaluation pipeline.
What format does the DOCX file need to be in?
The tool expects Word documents with sections marked as SECTION N --- Title, individual tests as Test N, User Prompt: fields, Expected Behavior: bullet lists, and optional Pass if: / Fail if: criteria.
Can I use this as a Python library instead of CLI?
Yes. Import convert_docx from docx_to_json_tool and call result = convert_docx("my_tests.docx"). It returns a dict with status, file paths, parsed data, and all validation reports. You can also import individual functions like parse_test_cases_from_docx, generate_jsonl, and run_generic_test_suite.
What is the JSONL format used for Azure AI Foundry?
Each line is a self-contained JSON object with test_id, section, user_prompt, expected_behavior (array), and optional pass_criteria, fail_criteria, and precondition fields. Upload this file directly to AI Foundry's evaluation framework.
Does it work with Microsoft Co-Pilot Studio test cases?
Yes. The tool supports any structured test-case document following the expected DOCX format, including those authored for Microsoft Co-Pilot Studio, Power BI AI agents, and Azure AI Foundry.
How many validation checks are run?
Approximately 45 automated QA checks across 11 categories: JSON structure, completeness, field validation, expected behavior content, special fields, AI Foundry compatibility, data type integrity, section integrity, prompt quality, delta-diff, and cross-reference consistency.
