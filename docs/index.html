Features
Everything in one pipeline
Parse, convert, validate, and report -- a single command does it all.
✎

Smart DOCX Parser

State-machine parser extracts sections, prompts, expected behaviors, pass/fail criteria, and preconditions from any structured Word document.

⧉

Dual Output

Generates hierarchical .json and flat .jsonl (one test per line) for direct Azure AI Foundry evaluation upload.

✓

45+ QA Checks

Embedded test suite validates structure, completeness, field integrity, prompt quality, and cross-reference consistency across 11 categories.

⇼

Delta-Diff Engine

Compares source DOCX against JSON output to catch prompt drift, missing entries, and keyword loss with similarity scoring.

⇄

Regression Detection

JSON-to-JSON comparator detects drift between conversion runs -- metadata, prompts, behaviors, and structural changes.

⚙

CLI + Library

Use from the command line or import convert_docx() directly into your Python code. Zero config, zero setup.

Get started
Two ways to use it
Command line for quick conversions. Python library for integration into your workflow.
Bash CLI Mode
# Install
$ pip install python-docx

# Convert (full QA pipeline)
$ python docx_to_json_tool.py convert tests.docx

# Validate existing JSON
$ python docx_to_json_tool.py validate src.docx out.json

# Compare two outputs
$ python docx_to_json_tool.py compare v1.json v2.json
Python Library Mode
from docx_to_json_tool import convert_docx

# Full pipeline, one call
result = convert_docx("my_tests.docx")

print(result["status"])      # "PASS"
print(result["json_path"])   # output path
print(result["jsonl_path"])  # JSONL path

# Silent mode for pipelines
result = convert_docx("t.docx", quiet=True)
FAQ
Common questions
Everything you need to know about the tool.
How do I convert a Word document to JSON for AI testing?
Install with pip install python-docx, then run python docx_to_json_tool.py convert your_test_cases.docx. The tool parses structured test cases and outputs validated JSON and JSONL files ready for Azure AI Foundry or any AI agent evaluation pipeline.
What format does the DOCX file need to be in?
Sections marked as SECTION N --- Title, individual tests as Test N, User Prompt: fields, Expected Behavior: bullet lists, and optional Pass if: / Fail if: criteria. See the GitHub README for the full format spec.
Can I use this as a Python library instead of CLI?
Yes. Import convert_docx from docx_to_json_tool and call result = convert_docx("my_tests.docx"). It returns a dict with status, file paths, parsed data, and all validation reports. Individual functions are also importable for custom workflows.
Does it work with Microsoft Co-Pilot Studio test cases?
Yes. Any structured test-case Word document works, including those authored for Microsoft Co-Pilot Studio, Power BI AI agents, and Azure AI Foundry evaluation.
How many validation checks are run?
Approximately 45 automated QA checks across 11 categories: JSON structure, completeness, field validation, expected behavior content, special fields, AI Foundry compatibility, data type integrity, section integrity, prompt quality, delta-diff, and cross-reference consistency.
Is the parser deterministic?
Yes. Running the parser twice on the same DOCX produces byte-for-byte identical output. This is verified by the idempotency test suite and is critical for CI/CD pipeline reliability.
